# 担当者引き継ぎプロトコル (Handover Protocol)

**マニフェスト価値:** 継続性 (Continuity)
**優先度:** 高

---

## トリガー条件

- 担当相談員の異動・退職
- 担当変更の申し出
- 「引き継ぎ資料を作りたい」

---

## 実行手順

### ステップ1：引き継ぎサマリーの生成

対象クライアントの包括的な引き継ぎ資料を生成する。

**生活保護受給者の場合（port 7688）:**
→ `livelihood-support` スキルのテンプレート10（引き継ぎサマリー）を `neo4j-livelihood` MCP の `read_neo4j_cypher` で実行。

このテンプレートは以下の情報を優先順位付きで提供する：
1. ⚠️ 避けるべき関わり方（NgApproach） ← 新担当者が最初に知るべき
2. ⚠️ 経済的リスク（EconomicRisk）
3. 🏥 精神疾患の状況
4. ✅ 効果的だった関わり方
5. 💪 発見された強み
6. 💰 金銭管理状況と支援サービス
7. 🤝 連携している機関

**障害福祉クライアントの場合（port 7687）:**
→ `neo4j-support-db` スキルのテンプレート2（クライアントプロフィール）を `neo4j` MCP の `read_neo4j_cypher` で実行。

### ステップ2：支援記録の要約

過去の支援記録から重要なパターンを抽出する。

→ `neo4j-support-db` スキルのテンプレート5（支援記録取得）およびテンプレート6（ケアパターン発見）を `neo4j` MCP の `read_neo4j_cypher` で実行。

**要約に含めるべきポイント:**
- 効果的だった対応トップ3
- 逆効果だった対応（禁忌として共有）
- 最近の変化・傾向
- 未解決の課題

### ステップ3：引き継ぎ資料の構成

以下の順序で資料を構成する（新担当者の安全を最優先）。

```markdown
# 引き継ぎ資料：[クライアント名]
作成日: [日付]
前任者: [前任者名]

## 1. ⚠️ 最初に読んでください（安全情報）
- 禁忌事項一覧
- 避けるべき関わり方
- 経済的リスク

## 2. ✅ 効果的な関わり方
- 実証済みの対応パターン
- 好きなこと・落ち着くもの

## 3. 📋 基本情報
- 氏名、年齢、特性
- 生育歴の要約
- 本人の願い

## 4. 🔗 支援体制
- キーパーソン（優先順位付き）
- 利用中のサービス・事業所
- 連携機関と担当者

## 5. 📅 期限管理
- 手帳・受給者証の更新日
- 次回モニタリング予定

## 6. 🔄 親の機能と代替体制
- 親が担うタスク（CareRole）
- 各タスクの代替手段
- バックアップの欠損

## 7. 📝 最近の支援状況
- 直近の支援記録要約
- 進行中の課題
- 申し送り事項
```

### ステップ4：エコマップの生成（オプション）

視覚的な支援関係図を引き継ぎ資料に添付する場合、ecomap-generatorスキルを使用。

テンプレート: `handover`（引き継ぎ用）を指定する。

### ステップ5：更新期限の確認

引き継ぎ時に期限切れ・期限間近の証明書がないか確認する。

→ `neo4j-support-db` スキルのテンプレート4（更新期限チェック）を `neo4j` MCP の `read_neo4j_cypher` で実行。

期限切れが発見された場合は、引き継ぎ資料の冒頭に赤字で警告を追加する。

---

## マニフェストの「継続性（Continuity）」の実現

このプロトコルは、マニフェスト第3の価値「継続性」を直接実現するものである。

- **暗黙知の形式知化:** 前任者の頭の中にあった「あの人にはこう接すると良い」を、NgAction・CarePreference・EffectiveApproachとしてデータベースに蓄積する
- **断絶の防止:** 担当者が変わっても、データベースに蓄積された知識がケアの質を維持する
- **コンテキストの継承:** 単なるデータの引き継ぎではなく、「なぜそうなのか」の文脈も含めて伝える

---

## 複数クライアントの一括引き継ぎ

担当者が複数のクライアントを引き継ぐ場合：

1. `neo4j-support-db` スキルのテンプレート1（クライアント一覧）で担当クライアント一覧を取得
2. 各クライアントについて上記手順を実行
3. 緊急度（禁忌事項の多さ、更新期限の近さ）でソートして優先順位をつける
4. PDF/Excelでの一括出力が必要な場合は `html-to-pdf` スキルまたは `xlsx` スキルを使用
