# プライバシーガイドライン & 個人情報保護運用規程

---

## 1. はじめに

nest-support は、知的障害・精神障害のある方の支援情報を管理するシステムです。扱うデータには**要配慮個人情報**（障害の種類・等級、医療情報、生育歴等）が多く含まれるため、個人情報保護法および関連ガイドラインに準拠した運用が必要です。

本ガイドラインは、nest-support を導入する事業所が、個人情報保護規程を整備する際の参考資料として作成されたものです。各事業所の既存規程と照らし合わせ、必要に応じてカスタマイズしてください。

---

## 2. データの保管場所と流れ

### 2.1 データの保管

nest-support のデータは**すべてローカル環境**（利用者のパソコン内の Docker コンテナ）に保管されます。クラウドへの自動送信は行われません。

```
[パソコン内]
├── Neo4j コンテナ (port 7687) ← 障害福祉データ
├── Neo4j コンテナ (port 7688) ← 生活困窮者自立支援データ
└── neo4j_data/, neo4j_livelihood_data/ ← 永続化ボリューム
```

### 2.2 Claude Desktop を通じた通信

Claude Desktop を使用する際、**入力内容は Anthropic 社のサーバーに送信されます**。これは Claude が応答を生成するために必要な通信です。

**対策:**
- 個人情報を含むデータの入力は、仮名化機能（後述）の利用を推奨
- 仮名化が困難な場合は、組織の個人情報保護規程に基づき利用可否を判断
- Anthropic 社のプライバシーポリシーを確認し、データの取り扱いを理解すること

### 2.3 LINE 通知（SOS サービス）

SOS 緊急通知機能は LINE Messaging API を使用します。LINE に送信される情報は最小限（クライアント名、緊急レベル）に制限されていますが、LINE 社のサーバーを経由します。

**対策:**
- SOS 通知に含める情報は必要最小限にとどめる
- LINE グループのメンバー管理を適切に行う
- 退職者のグループからの即時除外を徹底する

## 3. 仮名化機能の運用

### 3.1 仮名化とは

仮名化（Pseudonymization）とは、個人を直接特定できる情報（氏名、生年月日、電話番号等）を、識別記号や部分マスクに置き換える処理です。nest-support では、**表示時のみ**仮名化を行い、データベース内の実データは変更しません。

### 3.2 仮名化モード

| モード | 説明 | 用途 |
|--------|------|------|
| `off` | 仮名化なし | 通常の業務運用 |
| `mask` | 部分マスク（山田→山●●●） | 研修、説明会、デモ |
| `pseudonym` | 架空名に置換（山田→青山あおい） | テスト、開発、公開資料 |

### 3.3 仮名化の対象フィールド

| フィールド種別 | mask モード例 | pseudonym モード例 |
|--------------|-------------|-------------------|
| 人名 | 山●●● | 青山あおい |
| 電話番号 | 090-\*\*\*\*-5678 | 090-0000-0001 |
| 生年月日 | 1995-XX-XX | 1995-07-22 |
| 住所 | 東京都●●区●●● | ○○県桜ヶ丘1-1-1 |
| 医療機関名 | ●●病院 | さくら病院 |
| 組織名 | ●●事業所 | あさひ福祉協議会 |

### 3.4 仮名化の有効化手順

```bash
# 1. マイグレーション（clientId, displayCode, Identity ノードを追加）
uv run python scripts/migrate_pseudonymization.py --dry-run  # まず確認
uv run python scripts/migrate_pseudonymization.py             # 実行

# 2. .env に設定
PSEUDONYMIZATION_ENABLED=true
PSEUDONYMIZATION_MODE=mask          # または pseudonym
PSEUDONYMIZATION_SEED=any-string    # セッション固定シード

# 3. サービス再起動
docker compose restart
```

### 3.5 仮名化の安全上の例外

以下の情報は、**安全に直結するため仮名化の対象外**です:

- **NgAction（禁忌事項）**: リスクレベル、行為内容、理由
- **CarePreference（推奨ケア）**: ケアの指示内容
- **Certificate（手帳情報）**: 種類、等級（氏名は仮名化対象）

これは、緊急時に正確な禁忌情報やケア情報にアクセスできることが、利用者の安全を守るために不可欠だからです。

## 4. アクセス管理

### 4.1 物理的アクセス管理

nest-support はローカル環境で動作するため、パソコンへの物理的アクセス管理が重要です。

- パソコンのログインパスワードを適切に設定する
- 離席時は画面ロックを徹底する
- パソコンを共有する場合は、OS のユーザーアカウントを分離する

### 4.2 Neo4j データベースへのアクセス

デフォルトのパスワード（`password`）は本番運用前に必ず変更してください。

```bash
# docker-compose.yml の環境変数を変更
NEO4J_AUTH=neo4j/your_strong_password
```

### 4.3 ネットワークアクセス

Neo4j はデフォルトで `localhost` のみリッスンします。同一ネットワーク内の他のパソコンからアクセスする場合は、ファイアウォール設定とアクセス制御を適切に行ってください。

## 5. バックアップと廃棄

### 5.1 バックアップ

```bash
# 定期バックアップ（推奨: 週次）
docker compose stop
cp -r neo4j_data neo4j_data_backup_$(date +%Y%m%d)
cp -r neo4j_livelihood_data neo4j_livelihood_data_backup_$(date +%Y%m%d)
docker compose start
```

バックアップデータにも個人情報が含まれます。暗号化したUSBや施錠できる場所に保管してください。

### 5.2 データの廃棄

利用者が退所・転出した場合や、保存期間を超過した場合のデータ廃棄:

```bash
# 特定クライアントのデータを完全削除（Neo4j MCP 経由）
# ※ この操作は取り消せません
MATCH (c:Client {name: "氏名"})-[r]->(n)
DETACH DELETE c, n
```

廃棄の記録は監査ログ（AuditLog）に自動記録されます。

## 6. 研修・説明会でのデモ

### 6.1 デモデータの使用（推奨）

研修や説明会では、実際の個人情報を使わず、デモデータを使用してください。

```bash
# デモデータの投入
./installer/load-demo-data.sh

# デモ終了後にデモデータを削除
./installer/load-demo-data.sh --remove
```

デモデータは `isDemo = true` フラグが付与されており、実データと区別できます。

### 6.2 仮名化モードでの実データ使用

デモデータでは不十分な場合（実際の業務フローを見せたい場合）は、仮名化モードを有効にして実データを使用できます。

```
# .env を一時的に変更
PSEUDONYMIZATION_ENABLED=true
PSEUDONYMIZATION_MODE=mask
```

研修終了後は `PSEUDONYMIZATION_ENABLED=false` に戻してください。

## 7. インシデント対応

### 7.1 個人情報漏えい時の対応

1. **即座にアクセスを遮断**: Docker コンテナを停止 (`docker compose stop`)
2. **影響範囲の確認**: 監査ログ（AuditLog）で操作履歴を確認
3. **関係者への報告**: 管理者、個人情報保護責任者に報告
4. **原因の特定と再発防止**: パスワード変更、アクセス権限の見直し
5. **本人への通知**: 個人情報保護法に基づき、本人に通知

### 7.2 監査ログの活用

nest-support は全ての操作を AuditLog ノードに記録しています。

```
「最近の監査ログを見せて」
「山田さんに関する変更履歴を確認」
「田中さんが行った操作一覧」
```

## 8. 法的根拠

### 8.1 関連法令

- **個人情報の保護に関する法律**（個人情報保護法）
- **障害者の日常生活及び社会生活を総合的に支援するための法律**（障害者総合支援法）
- **生活困窮者自立支援法**
- **社会福祉法**

### 8.2 要配慮個人情報

nest-support が扱う以下の情報は「要配慮個人情報」に該当します:

- 障害の種類・等級
- 医療情報（かかりつけ医、処方薬等）
- 精神疾患に関する情報
- 生育歴・生活歴

要配慮個人情報の取得には、原則として**本人の同意**が必要です（個人情報保護法第20条第2項）。ただし、障害福祉サービスの提供に必要な場合は、法令に基づく例外が適用される場合があります。

### 8.3 第三者提供

nest-support のデータを他の事業所や機関に提供する場合は、原則として本人の同意が必要です。ただし、以下の場合は例外:

- 法令に基づく場合（虐待通報等）
- 人の生命・身体の保護に必要な場合
- 公衆衛生の向上に必要な場合
