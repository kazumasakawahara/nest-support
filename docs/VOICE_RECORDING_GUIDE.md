# 声の記録マニュアル — スマホで話すだけで支援記録

> **対象**: グループホーム・作業所の現場スタッフ
> **目的**: スマホに向かって話すだけで、AIが最高品質の構造化データを作成できるようにする

---

## なぜ「声の記録」が大切なのか

手書きやPC入力だと、どうしても記録が後回しになったり、大事な情報が抜け落ちます。
スマホに**その場で話すだけ**なら、感情や状況が新鮮なうちに記録でき、
AIが自動で「いつ」「誰が」「何が起きて」「どう対応して」「どうなったか」を整理してくれます。

---

## 基本の話し方テンプレート

### 「5W1E」で話す

以下の6つを意識するだけで、AIが正確に構造化できます。

| 項目 | 話す内容 | 例 |
|------|---------|-----|
| **いつ** (When) | 日付・時間帯 | 「今日の昼食時」「3月30日の午前中」 |
| **誰が** (Who) | クライアント名 | 「山本翔太さん」 |
| **何が** (What) | 何が起きたか | 「工事の音でパニックになった」 |
| **どうした** (Action) | あなたの対応 | 「別室に移動させた」 |
| **どうなった** (Effect) | 結果・効果 | 「10分後に落ち着いた」「効果的だった」 |
| **本人の気持ち** (Emotion) | 感情の様子 | 「怒っていた」「不安そうだった」「嬉しそうだった」 |

### 話し方の例

**良い例:**
> 「今日の午前中、山本翔太さんが作業中に隣のフロアの工事音でパニックになりました。
> 大声を上げて作業道具を投げたので、すぐに別室に移動させました。
> 別室では10分ほどで落ち着いて、その後は折り紙に取り組めました。
> 最初はすごく怒っていましたが、別室移動後は穏やかでした。
> 最近、工事が始まってからイライラが増えている気がします。」

AIがこの音声から抽出する情報:
- **状況**: 作業中
- **きっかけ**: 大きな音（工事音）
- **対応**: 別室に移動
- **効果**: Effective
- **感情**: Anger → Calm
- **文脈**: 隣のフロアの工事が原因、最近イライラが増えている

## 場面別テンプレート

### パニック・トラブル発生時

> 「[日時]、[名前]さんが[状況]で[何が起きたか]。
> 原因は[きっかけ]だと思います。
> [あなたの対応]をしたところ、[結果]でした。
> 本人は[感情の状態]でした。」

**具体例:**
> 「今日の昼、山本さんが食堂で大きな音に驚いてパニックになりました。
> 工事の振動が原因だと思います。
> 静かな個室に移動して食事を出したところ、落ち着いて食べられました。
> 最初は怖がっていましたが、個室では安心した様子でした。」

### 日常の穏やかな記録

> 「[日時]、[名前]さんは[活動]をしました。
> [様子の描写]。
> [感情の状態]でした。」

**具体例:**
> 「今日の午後、山本さんは折り紙の作業に取り組みました。
> 新しい作品を完成させて、誇らしそうに見せてくれました。
> とても嬉しそうでした。」

### ケアが効果的だった時

> 「[名前]さんに[ケアの内容]をしたら、[効果]でした。
> きっかけは[状況]で、本人は[感情]でした。
> これは今後も使えそうです。」

**具体例:**
> 「山本さんが作業中にイライラし始めたので、別室に移動して作業してもらったら、
> すぐに落ち着いて集中して取り組めました。
> 工事の音が原因で、最初は怒っていましたが、別室では穏やかでした。
> 別室移動はもう4回目ですが、毎回効果的です。」

## AIが喜ぶ話し方のコツ

### 1. 感情を具体的に

| こう話すと... | AIの理解 |
|-------------|---------|
| 「怒っていた」「イライラしていた」 | Anger |
| 「泣いていた」「元気がなかった」「落ち込んでいた」 | Sadness |
| 「怖がっていた」「おびえていた」 | Fear |
| 「不安そうだった」「そわそわしていた」 | Anxiety |
| 「嬉しそうだった」「笑顔だった」「楽しんでいた」 | Joy |
| 「穏やかだった」「落ち着いていた」 | Calm |
| 「びっくりしていた」「驚いていた」 | Surprise |

### 2. きっかけを明確に

| こう話すと... | AIのタグ付け |
|-------------|------------|
| 「工事の音で」「大きな音で」 | 大きな音 |
| 「新しい人が来て」「利用者が変わって」 | 環境変化 |
| 「他の人とのトラブルで」「喧嘩で」 | 人間関係 |
| 「お腹が痛いと」「体調が悪くて」 | 体調不良 |
| 「予定が変わって」「スケジュール変更で」 | スケジュール変更 |
| 「作業中に」「食事の時に」「入浴の時に」 | 状況タグ（作業中、食事、入浴） |

### 3. 「最近の変化」を一言添える

AIは「最近」の傾向変化を特に重視して分析します。
一言添えるだけで、予測精度が大きく向上します。

> 「**最近、工事が始まってからイライラが増えている気がします**」
> 「**先週はこんなことなかったのに、今週になって食欲が落ちています**」
> 「**この対応は3回目ですが、毎回うまくいっています**」

## 録音時の注意事項

### やること
- **静かな場所**で録音する（本人がいない場所で）
- **本人の名前**を必ず言う
- **対応の結果**（効果的だったか、ダメだったか）を必ず言う
- **できるだけ早く**記録する（その日のうちに）

### やらないこと
- 本人の前で録音しない（プライバシー配慮）
- 他の利用者の名前を出さない（「〇〇さん」ではなく「他の利用者」と言う）
- 録音ファイルを外部に送らない（端末内またはセキュアな共有フォルダのみ）

## 前提条件

音声や画像からの構造化登録には **Gemini API キー** が必要です。
管理者が `.env` ファイルに `GEMINI_API_KEY` を設定していることを確認してください。

```bash
# .env ファイルの確認（管理者が設定済みのはず）
grep GEMINI_API_KEY .env
```

未設定の場合は管理者に依頼してください。Gemini API キーは [Google AI Studio](https://aistudio.google.com/apikey) から無料で取得できます。

## 録音したら

録音した音声ファイル（.m4a, .mp3 等）は、以下の方法で登録できます:

### 方法1: Claude に直接依頼
> 「この音声ファイルを山本翔太さんの支援記録として登録してください」

### 方法2: 一括インポーター
```bash
uv run python scripts/multi_importer.py 録音ファイル.m4a --client "山本翔太" --supporter "鈴木"
```

### 方法3: フォルダ一括
```bash
uv run python scripts/multi_importer.py ./今日の記録/ --client "山本翔太"
```

## よくある質問

**Q: 何分くらい話せばいいですか？**
A: 30秒〜2分で十分です。長くても構いませんが、要点を押さえれば短くてOKです。

**Q: うまく話せなかったらどうなりますか？**
A: AIは多少のつまりや言い直しも理解できます。完璧に話す必要はありません。

**Q: 毎回「5W1E」を全部言わないとダメですか？**
A: 最低限「誰が」「何が起きたか」「どう対応したか」の3つがあれば記録になります。
でも「感情」と「きっかけ」を加えると、予測アラートの精度が大きく上がります。

**Q: 日本語以外でも大丈夫ですか？**
A: 日本語が最も精度が高いです。

**Q: 写真やメモも使えますか？**
A: はい。手書きメモの写真も読み取れます。同じインポーターで処理できます。
