# nest-support 全国配布戦略 & マイルストーンプラン

> 作成日: 2026-03-05
> 対象: 知的障害者の相談支援専門員、親の会、障害福祉サービス事業所
> 配布形態: 無償配布（導入支援付き）

---

## 1. 現状分析

### 1.1 プロダクトの強みと課題

**強み:**
- 「親亡き後」問題に特化した唯一の構造化データベース
- Claude AI による自然言語インターフェース（SQL不要）
- 緊急時プロトコル、SOS通知など実務直結の機能
- グラフデータベースによる支援ネットワークの可視化
- 弁護士が設計した権利擁護の視点

**配布に向けた課題:**
- Docker + Neo4j + Claude Desktop の技術スタックは福祉現場には高い壁
- Claude Desktop のサブスクリプション費用（月額 $20〜）が利用者負担
- 個人情報を扱うためセキュリティ・プライバシーへの配慮が不可欠
- 福祉業界のICTリテラシーは向上途上（タブレット普及率は増加傾向だが、AIツール導入はまだ黎明期）

### 1.2 対象ユーザーの特性

| ユーザー層 | 人数規模 | ITリテラシー | 主なニーズ |
|-----------|---------|-------------|-----------|
| 相談支援専門員 | 全国約4万人 | 中程度（記録システムは使用） | ケース管理、引き継ぎ、緊急対応 |
| 親の会（育成会等） | 全国約1,500団体 | 低〜中 | 親亡き後の備え、情報共有 |
| 障害福祉サービス事業所 | 全国約14万事業所 | 中程度 | 利用者情報管理、ケア記録 |

### 1.3 外部環境・追い風

- **令和8年度 障害福祉分野の介護テクノロジー導入支援事業**: 厚生労働省による補助金（補助率 10/10）が活用可能。ICTツール導入のインセンティブとなる
- **障害者総合支援法の改正**: 相談支援の質の向上・データ活用が政策的に推進されている
- **AI活用への社会的関心**: ChatGPT以降、福祉分野でもAI活用への関心が急速に高まっている

## 2. 配布に向けた仕様改善（ユーザビリティ）

### 2.1 導入障壁の解消（Phase A: 必須）

現在の技術スタック（Docker + Neo4j + Claude Desktop + MCP設定）は専門家向けであり、福祉現場での自律的導入は困難。以下の改善を行う：

#### A-1: ワンクリックインストーラーの開発
- **macOS**: `.dmg` パッケージ（Docker Desktop + Neo4j + 初期設定を自動化）
- **Windows**: `.exe` インストーラー（同上）
- インストール完了後、ブラウザで `localhost:7474` を開いてデモデータを確認できる状態にする

#### A-2: セットアップウィザード（対話型初期設定）
- 既存の `onboarding-wizard` スキルを拡張
- 「組織名」「担当者名」「最初のクライアント登録」を対話形式で案内
- サンプルデータ（匿名化済み）のプリロードオプション

#### A-3: Claude Desktop 設定の自動化
- `claude_desktop_config.json` の自動配置スクリプト
- MCP サーバー接続の自動テスト機能

### 2.2 運用しやすさの向上（Phase B: 推奨）

#### B-1: クイックスタートガイド（印刷用PDF）
- A4 4ページ以内の図解入り導入手順書
- 「今日から使える5つの操作」をステップバイステップで解説

#### B-2: 動画チュートリアル
- 5分×5本の短尺動画セット
  1. インストールと初期設定
  2. クライアント情報の登録
  3. 緊急時対応の使い方
  4. 訪問前ブリーフィングの活用
  5. 事業所検索と口コミ管理

#### B-3: FAQ & トラブルシューティング
- よくある質問集（Docker起動失敗、MCP接続エラー等）
- 問い合わせフォーム or メールサポート窓口の設置

### 2.3 セキュリティ・個人情報保護（Phase A: 必須）

#### C-1: 個人情報取扱ガイドライン
- データはすべてローカル保存（クラウド非送信）であることの明示
- Claude Desktop 利用時のデータ取扱いに関する説明文書
- 組織内の個人情報保護規程との整合性チェックリスト

#### C-2: 仮名化機能の強化
- `PSEUDONYMIZATION_ENABLED=true` 時のデモモード実装
- 研修・説明会で個人情報を使わずにデモができる仕組み

## 6. リスクと対策

| リスク | 影響度 | 対策 |
|-------|--------|------|
| Claude Desktop の料金が利用者にとって高い | 高 | 組織単位のライセンス交渉、ICT補助金の活用案内 |
| Docker/Neo4j の運用が困難 | 高 | ワンクリックインストーラー、リモートサポート |
| 個人情報漏洩への懸念 | 高 | ローカル完結アーキテクチャの説明、ガイドライン整備 |
| 継続的なメンテナンスリソース不足 | 中 | コミュニティ主導のサポート体制、ドキュメント充実 |
| Anthropic の API/製品仕様変更 | 中 | MCP標準への準拠、アーキテクチャの柔軟性確保 |
| 競合ツールの出現 | 低 | 「親亡き後」特化の独自性、ユーザーコミュニティの粘着性 |

## 8. ライセンス方針（案）

**推奨: MIT License**

理由：
- 福祉事業所が改変・再配布を自由に行える
- 商用利用も可能（将来的な事業化の選択肢を残す）
- コントリビューターが参加しやすい
- 法的にシンプルで福祉関係者にも理解しやすい

ただし、以下を `README.md` や利用規約に明記する：
- データベースに格納される個人情報の管理責任は利用者にあること
- Anthropic社の利用規約に従う必要があること
- 開発者は本ツールの利用により生じた損害について責任を負わないこと
